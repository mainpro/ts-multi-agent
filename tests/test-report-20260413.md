## 问题分析

### 问题1: 多任务识别 - LLM自主决策

**现象**: 用户输入"请假怎么申请报销呢"，只返回1个任务

**实际处理流程**:
```
[IntentRouter] ⚠️ 关键词命中多技能: time-management-qa, ees-qa (需要LLM决策)
[IntentRouter] 🤖 使用 LLM 匹配技能...
[IntentRouter] ✅ LLM 返回任务: 1个
```

**分析**:
- 系统正确地将多个技能提交给 LLM (time-management-qa, ees-qa)
- LLM 根据对话上下文分析，认为"请假怎么申请报销呢"是单一需求
- 这是 **LLM 自主决策**的结果，不是代码 bug

**不是 bug 的原因**:
1. 多任务识别依赖 LLM 理解能力
2. Prompt 中的规则是指导性的，不是强制性的
3. LLM 需要根据语义判断是否需要拆分任务

**测试验证**:
- "请假 GEAM 报销" → 正确返回3任务 ✅
- 这说明当需求明确分隔时，LLM 可以正确识别

**建议**: 
- 这是预期行为，由 LLM 自主判断是否需要多任务
