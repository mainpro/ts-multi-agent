# 报销提交问题处理

## 常见提交失败原因

| 原因类型 | 具体表现 | 处理方式 |
|---------|---------|---------|
| 发票格式问题 | "格式不支持" | 引导更换为PDF/XML/OFD |
| 附件问题 | 附件过大、格式不对 | 检查附件大小和格式 |
| 审批流程异常 | 系统报错、转交失败 | 转人工排查 |
| 金额异常 | 超出标准被驳回 | 告知报销标准 |
| 必填项缺失 | 某些字段未填写 | 提示补充填写 |

## 处理流程

### 第一步：识别问题

让用户发送报错截图，根据截图判断：
- 地址栏包含 ees.haier.net → EES系统问题
- 报错信息 → 判断具体原因

### 第二步：分类处理

**格式问题** → 见 invoice-format.md

**附件问题**
- 检查附件大小是否超过限制(通常10MB)
- 检查附件格式是否支持

**审批异常**
- 询问是否超过正常审批时间
- 尝试重新提交

**金额异常**
- 告知报销标准和限制
- 建议调整报销金额

### 第三步：解决或转人工

- 能解决的问题 → 提供解决方案
- 无法解决的问题 → 转人工

## 转人工话术

"好的，我这边已经记录您的问题了。由于涉及到系统后台排查，我帮您转接到人工客服处理，您稍等。"

## 历史记录查询

先查询用户历史记录，看是否有过类似问题及解决方案：
- 如有历史 → 复用历史方案
- 如无历史 → 按流程处理
