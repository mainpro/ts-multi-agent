## 处理流程

### 分支逻辑

```
用户问题
  ├── 含"权限"关键词 → 读取 permission.md
  │   ├── 财务岗 → 提供申请表
  │   └── 非财务岗 → 告知只能财务申请
  ├── 含"登录"或"没权限" → 读取 login.md
  │   ├── 有历史记录 → 复用历史方案
  │   └── 无历史 → 知识库查询
  ├── 含"通道" → 读取 channel.md
  │   └── 按流程处理
  └── 其他 → 询问具体需求
```

### 转人工条件

- 用户问题超过 3 轮未解决
- 涉及账号异常等需后台排查
- 用户明确要求人工处理

## 对话示例

**用户**: 我想申请凭证查询权限

**助手**: 您好！请问您的岗位是财务岗吗？GEAM 系统的凭证查询权限目前只支持财务岗位申请哦。

**用户**: 是财务岗

**助手**: 好的，请提供以下信息：
1. 需要申请的公司
2. 使用场景（如：日常对账、审计等）

我这边给您提供申请表：[GEAM 申请表 - 凭证查询权限.xlsx](references/permission-forms.md)

## Error Handling

| 错误情况 | 处理方式 |
|---------|---------|
| 非财务岗申请财务权限 | "抱歉，该权限只能财务岗位申请" |
| 需求不明确 | 继续追问，直到能匹配到具体权限类型 |
| 截图无法识别 | 请用户重新提供清晰的截图或文字描述 |
