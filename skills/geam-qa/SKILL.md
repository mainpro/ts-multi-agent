# GEAM 影像系统问答技能

## 系统概述

**系统名称**：集团全球电子影像档案管理中台
**系统简称**：GEAM
**主要功能**：管理海尔集团业务影像，包括凭证影像处理归档、银行电子回单归档、费用报账附件上传归档等。

## 执行步骤

### Step 1: 识别用户意图

根据用户描述判断需求类型：
- **权限申请** → 进入 Step 2
- **登录问题** → 进入 Step 3
- **提供签字文件** → 转人工处理

### Step 2: 权限申请处理 (分支逻辑)

**IF 用户说"我要申请权限"或类似表述:**
→ 反问："您好，请问您要登录系统操作什么呢？"

**根据用户回答匹配权限类型:**

| 关键词 | 权限类型 | 处理方式 |
|-------|---------|---------|
| 凭证查询/下载 | 凭证查询 | 确认是否财务岗，提供申请表 |
| 资产、UG、UB | 资产类扫描 | 提供申请表，需财务+直线领导签字 |
| 签收 | 签收权限 | 确认是否财务岗，提供申请表 |
| 回单 | 回单查询 | 提供申请表，需直线经理+财务部长会签 |
| 凭证后补 | 后补权限 | 确认是否财务岗，提供申请表 |
| 增加公司/新增公司 | 公司权限 | 提供申请表，需财务+直线领导签字 |

**IF 用户说"复制同事权限":**
→ 回复："抱歉系统目前不提供权限复制的功能，您可以问一下您同事之前在系统里都操作哪些工作，告诉我我来告诉您如何申请。"

### Step 3: 登录问题处理

**IF 用户截图显示"没有权限"或类似提示:**

反问："您好，请问您账号之前可以正常登录吗？"

**IF 用户回答"之前可以正常登录":**
→ 反问："您电脑是否登录过别人的账号？"
→ **IF 是:** 告知用户："请把电脑无感登录退出一下，清一下浏览器缓存再试试。"
→ **IF 否:** 进入权限申请流程（Step 2）

**IF 用户回答"之前没有登录过"或"没有正常登录":**
→ 进入权限申请流程（Step 2）

### Step 4: 提供申请表

根据匹配的权限类型，提供对应的申请表链接，并告知签字要求：

**通用回复模板：**
```
您好，开通相关权限需要填写申请表，并由相关人员签字后提交。

申请表下载链接：[对应申请表]

签字要求：[根据权限类型的签字要求]

请将填写好的申请表和签字文件发给我，我会转交给工程师处理。
```

## 常见场景示例

### 场景1：用户发送截图显示无权限
1. 识别截图中的"GEAM全球电子档案管理平台"标识
2. 按登录问题处理流程（Step 3）操作

### 场景2：用户直接说"我要申请权限"
1. 反问用户具体需求
2. 根据回答匹配权限类型
3. 提供对应申请表

### 场景3：用户提供签字文件
→ 直接转人工，由工程师处理

## Error Handling

| 错误情况 | 处理方式 |
|---------|---------|
| 用户非财务岗申请财务权限 | 告知"抱歉，该权限只能财务岗位申请" |
| 用户需求不明确 | 继续追问，直到能匹配到具体权限类型 |
| 用户提供的截图无法识别 | 请用户重新提供清晰的截图或文字描述 |
